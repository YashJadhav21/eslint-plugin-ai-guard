# Project Context: eslint-plugin-ai-guard

This document provides a comprehensive overview of the `eslint-plugin-ai-guard` project, detailing its purpose, architecture, current state, and future goals. Provide this entire document to any newly instantiated agent or developer so they understand the exact state and design philosophy of the project.

## 1. Project Goal
`eslint-plugin-ai-guard` is a specialized ESLint plugin designed to detect and prevent dangerous, buggy, or inefficient code patterns frequently generated by Large Language Models (LLMs) and AI coding assistants (like Copilot, ChatGPT, or Claude). 

Generative AI tools often prioritize generating code that "looks right" over code that is completely mathematically rigorous or secure. This plugin targets those specific "AI hallucinations" and common lazy patterns to ensure that AI-assisted code remains production-ready and secure.

### Core Philosophy
* **AST-Only**: Rules must rely purely on AST parsing. **No type-checking** is used. This enables zero-configuration, instant execution, and high performance, making the plugin very lightweight.
* **Low False Positives**: Generative AI flaws are usually structural. The AST patterns are tightly restricted to avoid complaining about legitimate human written code.

## 2. Technology Stack
* **Language:** TypeScript
* **Testing:** `vitest` partnered with ESLint's `@typescript-eslint/rule-tester`
* **Build System:** `tsup` — Configured to perform dual builds generating both CommonJS (CJS) and ECMAScript Modules (ESM), to support both the ESLint 8 format and the new ESLint 9 Flat Config (`eslint.config.js`).
* **Package Management:** `npm` (successfully published under `eslint-plugin-ai-guard@1.0.0`)
* **AST Framework:** `@typescript-eslint/utils` (used as the rigorous spine for AST node typing).

## 3. Implemented MVP Rules (Currently Active)

All 9 target MVP rules from Phase 2 have been actively developed, merged, and comprehensively covered by unit tests (174 total assertions).

### 🎯 Category: Error Handling
* **`no-empty-catch` (Error):** 
  * *Pattern Captured:* `try { ... } catch (e) {}`
  * *Why:* AI models often swallow errors silently by wrapping code blocks in empty try-catch parameters just to force them to run.
* **`no-broad-exception` (Warn):**
  * *Pattern Captured:* `catch(e: any)` or `catch(e: unknown)` without interior `instanceof` narrowings.
  * *Why:* Copilots lazily append `: any` to bypass TypeScript rules, effectively blinding developers to what actual error structurally failed.

### ⏱️ Category: Async Stability
* **`no-async-array-callback` (Error):**
  * *Pattern Captured:* `array.map(async () => {})` outside of `Promise.all`.
  * *Why:* AI frequently hallucinates that mapping async variables resolves them directly, rather than returning an array of raw promises. 
* **`no-floating-promise` (Error):**
  * *Pattern Captured:* Un-awaited async calls. Identified structurally either by naming conventions (`fetch`, `load`, `save`) or context.
  * *Why:* Forgets explicit `await` boundaries causing silent memory leaks and unhandled rejections locally.
* **`no-await-in-loop` (Warn):**
  * *Pattern Captured:* `const data = await fetch()` sequentially inside a `for...of` or `while` loop.
  * *Why:* O(N) network latency bottlenecks.

### 🛡️ Category: Security
* **`no-hardcoded-secret` (Error):**
  * *Pattern Captured:* Checking variable names (matching `password`, `key`, `token`) structurally holding massive hardcoded string hashes/literals.
  * *Why:* AI constantly hallucinates fake keys or worse, prints actual cached environment variables from prompt context directly into the written function body.
* **`no-eval-dynamic` (Error):**
  * *Pattern Captured:* `eval(user)` or `new Function($user)`.
  * *Why:* AI relies on highly insecure dynamic evaluations to solve algorithm edge cases.
* **`no-sql-string-concat` (Error):**
  * *Pattern Captured:* `SELECT * FROM x WHERE id = ${var}`.
  * *Why:* The AI refuses to use native ORM parameterized syntax cleanly at times and defaults to string concatenations resulting in SQL injection.
* **`require-auth-middleware` (Warn):**
  * *Pattern Captured:* Missing `protect`, `authenticate`, or local Passport middleware on `express.Router()` calls.
  * *Why:* Exposing administrative deletion endpoints publicly without context due to the LLM not realizing the application requires authentication context for it to work securely.

## 4. Current Configuration States
The plugin automatically exposes three native configuration environments mapping its definitions out of the box:
* `plugin:ai-guard/recommended` (balanced set of Warnings and Errors)
* `plugin:ai-guard/strict` (everything promoted aggressively to Error)
* `plugin:ai-guard/security` (focuses strictly on the security classification bounds)
